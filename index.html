Real-Time Jello

As part of a two-person team, I developed a real-time jello simulation using OpenGL and C++. My primary responsibilities included implementing the underlying physics and interactive behaviors. The simulation models the jello as a mass-spring system, utilizing both Euler and fourth-order Runge-Kutta integration methods for numerical stability and accuracy. To simulate translucency, we applied Beer's Law for light attenuation through the material. Performance was optimized using OpenMP for parallel computation.

Future Improvements:
- Implement per-vertex lighting to eliminate the appearance of flat, triangulated surfaces when the jello mesh is rendered at low resolution.
- Add a skybox or dynamic background to replace the solid color and better showcase the material properties and translucency of the jello.
- Accelerate the simulation using compute shaders by offloading physics calculations to the GPU for improved performance and real-time interactivity.
- Allow the jello to freely jiggle off the plate and interact dynamically with other objects within the bounding box.

Path Tracer

As part of a two-person team, we developed a CPU-based Monte Carlo Path Tracer in C++. The renderer supports loading and rendering of .obj models and simulates a variety of materials—including diffuse, specular, glossy, and glass—using bidirectional reflectance distribution functions (BRDFs). To improve performance, we implemented bounding volume hierarchies (BVH) for spatial acceleration and leveraged OpenMP for parallel processing. We conducted performance benchmarking across scenes with varying triangle counts and material properties, with detailed results available on our GitHub repository.

Future Improvements:
- Implement a built in denoiser to reduce the noise in our output
- Accelerate compute time using importance sampling and CUDA
